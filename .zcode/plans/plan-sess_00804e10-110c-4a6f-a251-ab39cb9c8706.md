2. **`GET /:id/calendar?days=28`** — Return per-day completion status for the last N days for one habit, in the shape the calendar UI needs: `[{ date: 'YYYY-MM-DD', status: 'done'|'partial'|'skipped' }]`. Reuses the `habit_completions` table.

5. **`DELETE /:id`** already exists (soft-archive) — wire to the Delete button.
6. **`PATCH /:id`** (or reuse `PUT`) — toggle `is_active` for the Pause/Resume button.
