## Add Commitment Duration + AI Evidence-Based Evaluation

### Overview
Extend the Add Habit flow so the user can declare how long they want to implement the habit (preset chips or custom start/end dates), and the AI evaluates their choice against evidence-based research (Lally et al. 2010 ‚Äî median 66 days to automaticity). The evaluation appears as an inline banner at the top of step 1 and persists through to the confirm step. Duration is optional ‚Äî if omitted, the AI recommends a default.

### 1. Backend ‚Äî `apps/backend/src/routes/ai.ts`

**Extend the `/parse-habit` route:**
- Accept optional `{ userInput, durationDays?, startDate?, endDate? }` in the body
- Compute `durationDays` from `startDate`/`endDate` if only dates are provided
- Pass the duration into the system prompt context

**Extend the system prompt** with a new DURATION EVALUATION rule and output field:
```
DURATION EVALUATION:
- If the user picked a duration, evaluate it against the evidence base:
  - Lally et al. (2010): median 66 days to automaticity (range 18‚Äì254)
  - Simple habits (e.g. drinking water) may automate in 20‚Äì40 days
  - Complex/identity-level habits (exercise, meditation) often need 80‚Äì100+ days
- Assess as "too_short" (< evidence floor), "appropriate", or "generous"
- Recommend a specific evidence-based duration and give one-line rationale
- If no duration chosen, recommend a default based on the behavior's complexity
```

**New output field in schema:**
```json
"duration_evaluation": {
  "user_chosen": number | null,
  "recommended_days": number,
  "assessment": "too_short" | "appropriate" | "generous",
  "rationale": "one-line evidence-based rationale",
  "milestone_note": "short note about what to expect at this duration"
}
```

### Flow summary
1. Step 0: user types habit description, optionally picks a duration chip (7/21/30/66/90/180/365) or custom start/end dates
2. Click Parse ‚Üí backend evaluates duration against evidence base
3. Step 1 opens with colored banner at top: AI verdict on the chosen duration + recommended alternative + milestone note
4. User proceeds through trigger/reward/friction/confirm with the banner persisting
5. On create: `target_days`, `start_date`, `end_date` persisted to DB
6. If no duration chosen: AI recommends an appropriate default, banner shows recommendation
