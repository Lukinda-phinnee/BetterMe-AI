<GUIDED_QUESTION>
{ "step":"wish", "question":"...", "suggestions":["...","..."], "placeholder":"..." }
</GUIDED_QUESTION>

The backend parses it, strips it from the visible reply, and returns it as a separate guidedQuestion field on the /coaching JSON response. The frontend renders the panel only when that field is present — so the AI alone controls when guided mode is active. No buttons, no client-side mode state.

This gives all four of your behaviors for free:

AI decides mode → presence of guidedQuestion is the signal.
Collapse between steps → panel closes on Confirm; reopens only if the next reply also carries guidedQuestion.
Mirrored to chat → the confirmed answer is sent through the normal message flow (persisted as a user message, shown in the transcript), so the existing history model is untouched.
Select then confirm → chip click fills the draft input; user edits/keeps it; Confirm locks it.
